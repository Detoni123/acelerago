## BUGS CORRIGIDOS

### 1. Espaçamento destaque/título na capa (canvas) — linha ~503
- **Problema:** `y += Math.round(52 * caLH) + 8` com `caLH=0` dava só 8px
- **Fix:** `y += Math.round(52 * (caLH > 1 ? caLH : 1)) + 36`
- **Motivo +36:** `textBaseline: alphabetic` — topo do texto fica `cap_height` acima do baseline (~55px para 76px de fonte). 36 garante ~25px de gap visual.

### 2. Cor do título da capa — canvas linha ~508 e React linha ~1163
- **Problema:** canvas e React usavam `slide.textColor` como fallback para cor do título
- **Fix canvas (linha ~508):** `const tc = slide.titleColor || '#ffffff'`
- **Fix React (linha ~1163):** `color:slide.titleColor||'#fff'` e `renderColored(slide.headline, slide.titleColor||'#fff', slide.accentColor||cp)`
- **Efeito:** mudar "Texto" não alterava mais cor do título no preview nem no export

### 3. Picker "Título" desacoplado do picker "Texto" — linha ~1286
- **Problema:** `val: slide.titleColor||slide.textColor||'#ffffff'` — picker mostrava `textColor` quando `titleColor=null`
- **Fix:** `val: slide.titleColor||'#ffffff'`
- **Efeito:** picker "Título" sempre mostra branco por padrão, independente de `textColor`

### 4. Gap imagem/título nos slides de conteúdo (canvas) — linha ~712
- **Problema:** `return y + imgH + 40` — 40px insuficiente; com `textBaseline: alphabetic` cap_height de ~55px causava overlap visual
- **Fix:** `return y + imgH + 80`

### 5. Fórmula de posicionamento da imagem no canvas — linha ~704
- **Problema:** `drawY = y - (drawH-imgH)/2 + offY*(imgH/100)` — direção OPOSTA ao CSS
- **Fix:** `drawY = y - (drawH-imgH) * (50 + offY) / 100`
- **Equivale ao CSS:** `backgroundPosition: center ${50+offY}%` (0%=topo, 50%=centro, 100%=baixo)
- **Antes:** `offY=-50` mostrava parte de baixo da imagem; agora mostra o topo

### 6. Canvas usa proporção natural da imagem — linhas ~699-704
- **Problema:** `drawH = imgH * sc` forçava altura de 380*scale, distorcendo aspecto da imagem
- **Fix:**
  ```javascript
  if (sc > 1) {
    drawW = containerW * sc;
    drawH = drawW * img.naturalHeight / img.naturalWidth;
  } else {
    const coverScale = Math.max(containerW / img.naturalWidth, imgH / img.naturalHeight);
    drawW = img.naturalWidth * coverScale;
    drawH = img.naturalHeight * coverScale;
  }
  ```
- **Espelha CSS:** `sc>1` → `backgroundSize:'${sc*100}%'`; `sc=1` → `backgroundSize:'cover'`

### 7. wrapText — texto cortado no fundo do canvas — linhas ~391, ~396
- **Problema:** texto sem limite passava de SH=1350px (sumia cortado)
- **Fix:** `if (cy < SH) ctx.fillText(...)` antes de cada linha

## PENDÊNCIA CRÍTICA: Imagens sumindo ao salvar/carregar

### Descrição do problema
Ao salvar o carrossel, as imagens (base64 em `slide.imageData`) são **intencionalmente removidas** do payload do banco (linha ~1710) e salvas apenas no `localStorage` com chave `carouselImg_<id>`. Ao recarregar, o sistema lê do localStorage primeiro. Se o localStorage for limpo, as imagens somem permanentemente.

### Localização do código
- **Save:** `gerador-carrossel.html` ~linha 1709-1720
  ```javascript
  const slidesForDB = slides.map(({ imageData, ...rest }) => rest); // strip
  // ...
  const saveImages = (id) => {
    const images = {};
    slides.forEach((s, i) => { if (s.imageData) images[i] = s.imageData; });
    LS.set(`carouselImg_${id}`, images);
  };
  ```
- **Load:** ~linha 1769-1780
  ```javascript
  const storedImages = LS.get(`carouselImg_${entry.id}`, {});
  const slidesWithImages = (entry.slides || []).map((s, i) =>
    storedImages[i] ? { ...s, imageData: storedImages[i] } : s
  );
  ```

### Por que não colocamos direto no banco
Tentamos `slidesForDB = slides` (com imageData). O Supabase rejeitou o payload (base64 de 2 imagens = 1-4MB). A update retornou array vazio, disparando o erro "Salvar falhou: sem permissão ou ID inválido."

### Solução correta (próxima sessão)
**Supabase Storage** — salvar cada imagem como arquivo e guardar a URL no slide:
1. `supabase.storage.from('carousel-images').upload(path, blob)`
2. Guardar `slide.imageUrl` em vez de `slide.imageData`
3. No load: buscar as URLs do storage
4. No export/canvas: carregar a imagem via URL antes de desenhar

**Tabela:** `saved_carousels` em Supabase
**Storage bucket sugerido:** `carousel-images` (criar como público ou signed URLs)

### Workaround atual
As imagens ficam no localStorage. Enquanto o usuário não limpar o cache e usar o mesmo browser, funciona. Para maior segurança, avisar o usuário que imagens só existem localmente.
